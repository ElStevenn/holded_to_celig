Introducción

Esta herramienta permite integrar automáticamente las facturas y documentos de Holded con el sistema de contabilidad Cegid. La sincronización se realiza de forma segura y permite migrar diferentes tipos de documentos.

¿Qué hace esta herramienta?
Exporta documentos (facturas, presupuestos, compras) desde Holded y los crea automáticamente en Cegid, manteniendo la numeración y datos correctos.

Flujo General

  • Conecta una cuenta de Holded mediante su API Key
  • Configura qué tipos de documentos quieres migrar
  • Define los offsets para continuar la numeración
  • Exporta las facturas con un solo clic
  • Monitorea el progreso en tiempo real

Conceptos Clave

API Key (Holded)

Es la clave de acceso a la API de Holded. Te permite conectarte a tu cuenta y obtener los documentos. Se obtiene desde el panel de Holded en:

Cómo obtenerla:
Holded → Configuración → API → Generar API Key
Código de Empresa (Cegid)

Identificador único de tu empresa en Cegid. Es el código que aparece en Cegid para tu empresa. Ejemplo: 72, 148

Tipos de Documentos a Migrar

Define qué tipos de documentos de Holded quieres exportar a Cegid. En la interfaz verás nombres descriptivos, pero internamente se usan códigos técnicos:

✅ Tipos Disponibles Actualmente
En la interfaz Código interno Descripción
Facturas de venta invoice Facturas emitidas a clientes
Presupuestos estimate Presupuestos enviados a clientes
Facturas de compra purchase Facturas recibidas de proveedores
🔒 Próximamente (Deshabilitados)
Tipo Estado
Pedidos de venta En desarrollo
Pedidos de compra En desarrollo
Notas de crédito En desarrollo
Albaranes En desarrollo
Remitos En desarrollo
Facturas proforma En desarrollo
Nota: Actualmente solo están disponibles los 3 primeros tipos de documentos. Los demás aparecen en el menú desplegable pero están deshabilitados hasta que se complete su implementación.
Offset (Número de Documento)

Es el último número de documento que se usó en Cegid para ese tipo. Sirve para continuar la numeración correctamente y evitar duplicados.

Ejemplo práctico:

Si en Cegid la última factura es FAC-116, el offset debe ser 116. La próxima factura que se cree será FAC-117.

Importante: Cada tipo de documento tiene su propio offset. Si migras facturas (invoice) y presupuestos (estimate), cada uno necesita su offset correspondiente.
Tipo de Cuenta

Define qué motor interno usa la integración para crear las facturas en Cegid:

  • nuevo_sistema (Recomendado) → Motor moderno y optimizado
  • normal → Motor legacy, usar solo si el nuevo no funciona

Crear Nueva Integración

Sigue estos pasos para configurar una nueva integración entre Holded y Cegid:

1 Haz clic en "Nueva"

En el panel principal, haz clic en el botón + Nueva en la sección "Cuentas Holded".

2 Completa los datos básicos

Nombre empresa: Un nombre descriptivo (ej: "Hermanos Pastor Vellisca SL")

API Key: La clave API obtenida desde Holded

Código empresa: El código de tu empresa en Cegid (ej: 72)

Offset documento: El último número de factura usado en Cegid

Tipo de cuenta: Selecciona nuevo_sistema (recomendado)

3 Configura los tipos de documentos

Añade los tipos de documentos que quieras migrar. Ahora puedes seleccionarlos desde un menú desplegable:

Ejemplo de configuración:
Tipo (seleccionable) Offset Descripción
Facturas de venta 116 Última factura: FAC-116
Presupuestos 69 Último presupuesto: PRE-69
Facturas de compra 64 Última compra: COM-64
Tip: Haz clic en "+ Añadir tipo de documento" para agregar más tipos. Ya no necesitas escribir códigos como "invoice" o "estimate", simplemente selecciona del menú desplegable.

4 Guarda la configuración

Haz clic en "Guardar" y la integración estará lista para usar.

¡La cuenta aparecerá en la tabla principal!

Tipos de Cuenta en Detalle

nuevo_sistema (Recomendado)

  • Motor moderno y optimizado
  • Mejor manejo de errores
  • Logs más detallados
  • Más rápido y estable

normal

  • Motor legacy (antiguo)
  • Usar solo si el nuevo sistema falla
  • Menos características
Recomendación: Siempre prueba primero con nuevo_sistema. Solo cambia a normal si encuentras errores.

Migración Automática

El sistema incluye una función de migración automática que exporta todas las facturas periódicamente sin intervención manual.

Configuración Actual:
Por defecto, la migración automática se ejecuta cada 30 días a las 00:00 (medianoche).

¿Cómo Funciona?

La migración automática utiliza Celery Beat (un programador de tareas) que se ejecuta en segundo plano y:

  • Procesa todas las cuentas de Holded configuradas
  • Exporta todos los tipos de documentos (facturas, presupuestos, compras)
  • Actualiza automáticamente los offsets
  • Registra todos los eventos en logs detallados
  • Continúa con la siguiente cuenta si una falla

Ver Estado en el Panel

En la página principal verás una tarjeta destacada que muestra:

  • Estado: Activo / Desactivado
  • Intervalo: Cada cuántos días se ejecuta (ej: 30 días)
  • Próxima ejecución: Cuándo se ejecutará la siguiente migración
  • Botón "Ejecutar Ahora": Para iniciar una migración manual inmediatamente

Ejecutar Migración Manual

1 Haz clic en "Ejecutar Ahora"

En la tarjeta de "Migración Automática" del panel principal, haz clic en el botón "Ejecutar Ahora".

2 La tarea se inicia

El sistema iniciará la migración inmediatamente en segundo plano. Verás una notificación de confirmación.

Nota: La migración se ejecuta de forma asíncrona, por lo que puedes seguir usando la aplicación mientras se procesa.

3 Revisa los logs

Los logs de la migración automática se guardan en el sistema. Puedes revisar qué se procesó y si hubo errores.

Configurar el Intervalo

El intervalo de migración se configura mediante variables de entorno: 

# En tu archivo .env o variables de entorno
AUTO_MIGRATION_ENABLED=true
AUTO_MIGRATION_INTERVAL_DAYS=30

# Para cambiar el intervalo, modifica el valor:
# Cada 15 días:
AUTO_MIGRATION_INTERVAL_DAYS=15

# Cada mes (aprox.):
AUTO_MIGRATION_INTERVAL_DAYS=30

# Cada 2 meses:
AUTO_MIGRATION_INTERVAL_DAYS=60
Importante: Después de cambiar las variables de entorno, debes reiniciar los servicios de Celery Beat para que los cambios surtan efecto: 
make restart-beat

Desactivar Migración Automática

Si deseas desactivar la migración automática temporalmente:

# En tu archivo .env
AUTO_MIGRATION_ENABLED=false

Y reinicia Celery Beat: make restart-beat

Ventajas:
• No necesitas recordar exportar facturas manualmente
• Mantiene tus sistemas sincronizados automáticamente
• Reduce errores humanos
• Ahorra tiempo de gestión

Exportar Facturas Manualmente

1 Abre la cuenta

Haz clic en el botón de edición en la cuenta que quieres exportar.

2 Inicia la exportación

Haz clic en el botón verde "Exportar facturas a Cegid".

El proceso comenzará inmediatamente y se ejecutará en segundo plano.

3 Monitorea el progreso

Se abrirá automáticamente una ventana con los logs en tiempo real donde podrás ver:

  • Qué documentos se están procesando
  • Errores (si los hay)
  • Estado de cada factura

4 Revisa el resultado

Al finalizar, verás un resumen del proceso. Revisa que todo se haya exportado correctamente.

Los offsets se actualizan automáticamente después de cada exportación.

Solución de Problemas

❌ Error: "API Key inválida"

Solución: Verifica que la API Key de Holded esté correcta y activa.

❌ Error: "Código de empresa no encontrado"

Solución: Revisa que el código de empresa en Cegid sea correcto.

❌ Error: "Factura duplicada"

Solución: El offset está mal configurado. Ajusta el offset al último número usado en Cegid.

❌ La exportación se detiene

Solución:

  • Revisa los logs para ver el error específico
  • Verifica la conexión a internet
  • Asegúrate de que los tokens de Cegid estén actualizados

⚠️ Algunas facturas no se exportan

Posibles causas:

  • Facturas ya existentes en Cegid
  • Datos incompletos en Holded
  • Clientes no sincronizados

Solución: Revisa los logs detallados para identificar qué facturas fallaron y por qué.

Consejo: Siempre revisa los logs después de una exportación. Te indicarán exactamente qué se procesó y si hubo algún problema.
